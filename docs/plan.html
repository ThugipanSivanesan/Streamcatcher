Build Plan

Streamcatcher

One-line goal: A cross-platform Python CLI that connects to an RTMP or RTSP stream and plays it (audio + video) in a small, positionable window — with snapshot capture and resilient reconnection — built securely from commit #1.

  • Backend: libVLC
  • Audio + video
  • Cross-platform
  • CLI: Typer

1. What we're building

streamcatcher is a command-line tool. You give it a stream URL; it opens a small native window and plays the live stream. It can grab still frames, resize/reposition its window, and it keeps trying to reconnect when a flaky stream drops.

Example usage:

streamcatcher play rtsp://camera.local:554/stream1
streamcatcher play rtmp://live.example/app/key --width 640 --height 360 --x 40 --y 40
streamcatcher play rtsp://cam/stream --snapshot shot.png      # grab one frame, exit
# while a window is open: press 's' to save a snapshot, 'q' to quit

Confirmed requirements

Decision Choice
Playback backend python-vlc (libVLC) — robust RTMP+RTSP, handles audio, manages its own window, reconnects well
Audio Audio + video
Platforms Cross-platform (macOS / Linux / Windows)
Features View, Snapshot (both a live hotkey and a one-shot --snapshot flag), Window size + position flags
Recording Out of scope for now
Reconnect Auto-retry with exponential backoff
CLI framework Typer

2. Why libVLC

  • Handles both RTMP and RTSP, plus their audio tracks, with no extra plumbing — the FFmpeg-based alternatives are video-only or need a separate audio path.
  • Manages its own decode + render + window, and tolerates real-world stream hiccups far better than cv2.imshow.
  • Cross-platform via a single, well-maintained binding (python-vlc).
  • Trade-off we accept: libVLC (the VLC runtime) must be present on the machine. We document install per-OS and detect+explain clearly if it's missing. Snapshots use libVLC's native video_take_snapshot, so they work even though we never touch raw frames ourselves.

3. Architecture (offline-first, pluggable)

Follows the workflow's provider pattern so the app and the entire test suite run with no network, no credentials, no live stream.

src/streamcatcher/
  __init__.py
  cli.py            # Typer app: `play` command, flag parsing, wires everything
  config.py         # pydantic-settings Settings; StreamUrl as SecretStr; log redaction seed
  logging_setup.py  # root logger + secret-redacting filter (from the workflow)
  player/
    base.py         # Player Protocol: play(), stop(), snapshot(path), is_playing()
    factory.py      # get_player(settings) keyed on a backend enum; default = offline stub
    vlc_player.py   # LIVE provider — lazy-imports `vlc` INSIDE the call path
    stub_player.py  # OFFLINE default — no window, deterministic, used by tests
  reconnect.py      # backoff policy (attempts, base delay, cap, jitter) — pure & unit-tested
  snapshot.py       # filename/timestamp helpers; one-shot vs hotkey orchestration
  errors.py         # typed exceptions (LibVlcNotFound, StreamUnavailable, ...)

Key design points

  • Player is a Protocol. factory.get_player() returns the offline stub by default; the live libVLC player is only selected when actually playing a real stream, and it lazy-imports vlc inside the method so importing the package never requires VLC to be installed.
  • The stream URL is a secret. RTSP/RTMP URLs routinely embed user:pass@host. The URL is stored as a SecretStr, never printed in full, and its value seeds the log-redaction filter — so credentials can't leak into logs or tracebacks.
  • Reconnect logic is pure and deterministic (compute the next delay from attempt count), so it's fully unit-testable without sleeping or connecting. The live player calls into it; safety default = give up cleanly after the cap rather than spin forever.
  • --snapshot one-shot path doesn't open the interactive window: it connects, waits for first frame, writes the file, exits — good for scripts/cron.
  • Hotkeys (s = snapshot, q = quit) are read on the window/console; the stub reports supported keys so tests can assert wiring without a GUI.

4. Security posture (from commit #1)

  • Stream URL + any credentials handled as SecretStr; secret_values() seeds a redacting log filter (also matches Bearer …, sk-…, long hex/base58 shapes).
  • .gitignore excludes .env, *.key, *.pem, secrets/; .env.example holds placeholders only.
  • Pre-commit: gitleaks, ruff (+fix), ruff-format, detect-private-key, large-file / EOF / whitespace / yaml / toml checks.
  • CI (GitHub Actions), PR-gated + weekly cron, least-privilege, all actions SHA-pinned:
  1. lint & test — ruff check + ruff format --check + pytest
  2. gitleaks (pinned CLI binary)
  3. osv-scanner (pinned CLI binary) against uv.lock
  • Dependabot monthly, grouped, for uv + github-actions.
  • No eval/exec/pickle/shell=True; the only subprocess-ish surface is libVLC via its Python binding. HTTP/stream operations get timeouts and fail closed.

5. Tooling & stack

  • Python 3.12+, packaged with uv + hatchling, src/ layout.
  • Runtime deps: python-vlc, typer, pydantic, pydantic-settings.
  • System dep: VLC / libVLC (documented per-OS; detected at runtime).
  • Dev: pytest + pytest-cov, ruff, pre-commit.
  • Lint: ruff line-length 100, select = E,F,I,B,UP,W, target-version = py312.
  • Entry point: streamcatcher = "streamcatcher.cli:app".

6. Delivery — small vertical slices (TDD, branch-per-feature, merge on green)

Each slice is one branch, tests first, merged only when all CI checks are green.

  1. Slice 0 — Secure bootstrap (to main). Repo skeleton, pyproject, ruff/pytest config, typed+redacted config, .gitignore/.env.example, pre-commit, CI (3 jobs), dependabot, README. Verified by pre-commit run --all-files clean; CI first gates on Slice 1's PR.
  2. Slice 1 — CLI skeleton + config. streamcatcher play <url> parses flags, builds Settings, selects the offline stub player, logs a redacted URL. Full test coverage, no network.
  3. Slice 2 — libVLC live player. vlc_player.py behind the factory, lazy-imported; opens the window, plays audio+video, honors --width/--height/--x/--y. Clear error if libVLC absent. Manual smoke script (not collected by pytest) against a public RTSP/RTMP test stream.
  4. Slice 3 — Reconnect with backoff. Pure backoff policy + wiring into the live player; unit tests cover the delay schedule and give-up behavior.
  5. Slice 4 — Snapshots. One-shot --snapshot PATH and the live s hotkey, via libVLC video_take_snapshot; timestamped default filenames. Tested against the stub.
  6. Slice 5 — Docs & polish. README quickstart, per-OS libVLC install notes, troubleshooting, --help examples.

7. How we verify

  • pytest green offline on every slice; a grep of the test run confirms zero real network/credential use.
  • Live paths validated by hand via an opt-in smoke script against free public RTSP/RTMP test streams — never in CI.
  • ruff check + ruff format --check clean; pre-commit run --all-files clean.
  • Manual end-to-end: open a real stream, confirm audio+video in the small window, take a snapshot, kill the source and watch it retry.
